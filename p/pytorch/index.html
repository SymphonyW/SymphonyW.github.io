第一章：环境搭建：保姆级安装教程

对于零基础的你来说，第一步就是把环境搭好，这是所有学习的基础。我们会覆盖 Windows、Linux、Mac 全平台，以及 CPU 和 GPU 版本的安装。

1.1 为什么要使用虚拟环境？

在安装 PyTorch 之前，强烈推荐你使用 Anaconda/Miniconda 来管理 Python 环境。它可以帮你：

  • 隔离不同项目的依赖，避免包版本冲突

  • 一键创建干净的环境，不会搞乱你的系统 Python

  • 方便的管理不同版本的 Python 和 CUDA

如果你还没有安装 Miniconda，可以先去官网下载，选择对应你系统的版本，一路默认安装即可。

第二章：核心基础：张量（Tensor）——PyTorch 的数字积木

环境搭好了，接下来我们来学习 PyTorch 最基础的概念：张量（Tensor）。

2.1 什么是张量？

简单来说，张量就是一个多维数组。你可以把它理解为升级版的 NumPy 数组，但是它有两个超能力：

  1. 可以在 GPU 上运行，加速计算

  2. 可以自动记录计算历史，帮你自动求导

我们平时接触的所有数据，最终都会变成张量，才能在 PyTorch 里处理。

维度 名称 生活中的例子
0 维 标量 一个单独的数字，比如 5、3.14
1 维 向量 一行数据，比如 [1,2,3,4]，就像你的成绩单
2 维 矩阵 一个表格，比如 Excel 的二维表格
3 维 3D 张量 一张彩色图片，形状是 (高, 宽, 通道)，比如 28×28 的灰度图是 (28,28,1)
4 维 4D 张量 一批图片，形状是 (批次, 通道, 高, 宽)，比如一次喂 32 张图片就是 (32,1,28,28)

import torch
import numpy as np

# ======== 从 Python 列表创建张量 ========
# 从列表创建1维张量
t1 = torch.tensor([1, 2, 3, 4])
print(t1)

# 从二维列表创建2维张量
t2 = torch.tensor([[1, 2], [3, 4]])
print(t2)

# 如果把tensor大写为Tensor，则会创建一个默认类型的张量，通常是torch.float32
t3 = torch.Tensor([1, 2, 3, 4])
print(t3)

print("==============================")

⚠️ 避坑提醒：永远用小写的 torch.tensor()，不要用大写的 torch.Tensor()！ 如果把tensor大写为Tensor，则会创建一个默认类型的张量，通常是torch.float32，这可能会导致你后续的计算出现类型不匹配的错误，尤其是当你想创建整数张量时。

# ======== 从 NumPy 数组创建张量 ========
# 创建一个 NumPy 数组
np_arr = np.array([[1, 2], [3, 4]])
print(np_arr)

# 转成 PyTorch 张量
t4 = torch.from_numpy(np_arr)
print(t4)

# 默认情况下，torch.from_numpy() 创建的张量与原始 NumPy 数组共享内存。这意味着修改张量会影响原始数组，反之亦然。
t4[0, 0] = 10  # 修改张量的值
print(t4)       # 张量被修改了
print(np_arr)   # NumPy 数组也被修改了，因为它们共享内存

# 如果不想共享内存，可以使用 torch.tensor() 来创建一个新的张量
t5 = torch.tensor(np_arr)
# 查看 t5 和 np_arr 的内存地址
print("t5 内存地址:", t5.data_ptr())
print("np_arr 内存地址:", np_arr.__array_interface__['data'][0])

💡 注意：默认情况下，NumPy 数组和张量是共享内存的！修改其中一个，另一个也会跟着变！ 如果你想要独立的副本，用 t3 = torch.tensor(np_arr)，这会做一个深拷贝。

第三章：自动微分（Autograd）——PyTorch 的灵魂

学完了张量，接下来我们来学习 PyTorch 最核心的功能：自动微分。这也是为什么 PyTorch 能这么方便的训练神经网络的原因。

3.1 什么是自动微分？

深度学习训练的核心就是反向传播，简单来说就是：我们计算出预测的误差（损失）之后，要把这个误差反向传给每一个参数，然后根据误差调整参数，让误差越来越小。

这个过程里最麻烦的就是求梯度（也就是求导），手动求导太麻烦了，而 PyTorch 的自动微分可以帮你自动计算所有参数的梯度，你只需要写前向传播的代码，剩下的求导 PyTorch 帮你搞定！

3.2 一个最简单的例子

我们来看一个最简单的例子，帮你理解自动微分：

import torch
import numpy as np

# 定义一个参数w，我们要对他求导，所以设置requires_grad = True
# requires_grad是PyTorch中一个非常重要的属性，它告诉PyTorch需要对这个张量进行梯度计算。只有设置了requires_grad=True的张量，PyTorch才会在反向传播过程中计算它的梯度。
w = torch.tensor(10.0, requires_grad=True)

# 前向传播：我们的损失函数是 loss = 2 * w²
loss = 2 * w ** 2

# 反向传播：自动计算梯度，
loss.backward()

# 查看 w 的梯度，也就是 d(loss)/dw，pytorch会自动计算这个值，并存储在 w.grad 中
print(w.grad)

我们手动算一下：对 loss=2w² 求导，导数是 4w，w 是 10，所以导数是 40，和 PyTorch 算出来的一模一样！

就这么简单！PyTorch 自动帮你完成了求导，你完全不用管中间的过程！

3.3 核心概念解释

  • requires_grad=True：告诉 PyTorch，这个张量是需要求梯度的，你要记录它的所有计算历史。

  • 计算图：当你做前向计算的时候，PyTorch 会在背后偷偷构建一个动态的计算图，记录所有的操作。

  • loss.backward()：触发反向传播，从损失开始，沿着计算图反向遍历，自动算出所有 requires_grad=True 的张量的梯度。

  • w.grad：计算出来的梯度会存在张量的 .grad 属性里，你可以直接用它来更新参数。

这是所有深度学习训练的核心模板，你一定要记住：

import torch.nn as nn

class MyNet(nn.Module):
    def __init__(self):
        super().__init__()      # 必须调用父类的初始化

        # 定义我们的类
        self.flatten = nn.Flatten()     # 展平层，把28x28的图片展平成728维
        self.fc1 = nn.Linear(28*28, 128) # 全连接层（线性层）：输入784，输出128
        self.fc2 = nn.Linear(128, 256)  # 全连接层（线性层）：输入128，输出256
        self.fc3 = nn.Linear(256, 10)   # 输出层：输入256，输出10（对应10个数字）

    def forward(self, x):
        # 前向传播的逻辑，x是输入的张量
        x = self.flatten(x)     # 先展平
        x = self.fc1(x)
        x = torch.relu(x)       # ReLU激活函数，加入非线性
        x = self.fc2(x)
        x = torch.relu(x)
        x = self.fc3(x)

        return x                # 返回输出

# 选择设备

# 定义一个简单的多层感知机（MLP）模型
class Model(nn.Module):
    def __init__(self):
        super().__init__()      # 必须调用父类的初始化
        self.flatten = nn.Flatten()     # 展平层，把28x28的图片展平成728维
        self.hidden1 = nn.Linear(28*28, 128) # 全连接层（线性层）：输入784，输出128
        self.hidden2 = nn.Linear(128, 256)  # 全连接层（线性层）：输入128，输出256
        self.out = nn.Linear(256, 10)   # 输出层：输入256，输出10（对应10个数字）

    def forward(self, x):
        # 前向传播的逻辑，x是输入的张量
        x = self.flatten(x)     # 先展平
        x = self.hidden1(x)
        x = torch.relu(x)       # ReLU激活函数，加入非线性
        x = self.hidden2(x)
        x = torch.relu(x)
        x = self.out(x)

        return x                # 返回输出
    

# 模型实例化，并且转移到GPU上
model = Model().to(device)
print(model)                    # 打印模型结构看看


# ---------------------- 3. 定义损失函数和优化器 ----------------------
# 损失函数：多分类任务用交叉熵损失
loss_fn = nn.CrossEntropyLoss()

# 优化器：用Adam优化器优化模型的参数，学习率是0.01
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)


# ---------------------- 4. 训练模型 ----------------------
def train(dataloader, model, loss_fn, optimizer):
    model.train()   # 设置模型为训练模式
    batch_num = 0   # 记录训练了多少个batch了

    for X, y in dataloader:   # 遍历每个batch的数据
        batch_num += 1
        X, y = X.to(device), y.to(device)   # 把数据转移到GPU上

        # 前向传播
        pred = model(X)    # 模型的预测结果
        loss = loss_fn(pred, y)  # 计算损失

        # 反向传播和优化
        optimizer.zero_grad()   # 清空之前的梯度
        loss.backward()         # 反向传播计算新的梯度
        optimizer.step()        # 更新模型参数

        if batch_num % 100 == 0:   # 每训练100个batch打印一次损失
            print(f"训练了 {batch_num} 个batch，当前损失：{loss.item():.4f}")


# ----------------------- 5. 测试模型 ----------------------
def test(dataloader, model, loss_fn):
    print("正在测试模型...")
    model.eval()    # 设置模型为评估模式
    size = len(dataloader.dataset)  # 测试集的样本数量
    num_batches = len(dataloader)   # 测试集的batch数量
    print(f"测试集样本数量：{size}，batch数量：{num_batches}")
    test_loss, correct = 0, 0       # 累计损失和正确的预测数量
    

    with torch.inference_mode():    # 自动关闭梯度计算，不需要计算梯度了，节省内存和计算
        for X, y in dataloader:     # 遍历每个batch的数据，X是输入，y是标签。为什么X要大写？因为它是一个张量，习惯上张量用大写字母表示。
            X, y = X.to(device), y.to(device)

            pred = model(X)         # 模型的预测结果，形状是 [batch_size, 10]，每行是一个样本的10个类别的预测分数
            test_loss += loss_fn(pred, y).item()  # 累加损失
            correct += (pred.argmax(1) == y).type(torch.float).sum().item()  # 累加正确的预测数量
            # pred.argmax(1) 是取每行最大值的索引，也就是预测的类别，和y比较是否相等，得到一个布尔张量，再转换成float类型，最后求和得到正确的数量

    test_loss /= num_batches  # 平均损失：总损失除以batch数量
    accuracy = correct / size  # 准确率：正确的数量除以总样本数量
    print(f"测试结果：测试集平均损失：{test_loss:.4f}，准确率：{accuracy:.4f}")
    return accuracy


# ----------------------- 6. 运行训练和测试 ----------------------
print("开始训练和测试模型...")
epochs = 10

for epoch in range(epochs):
    print(f"Epoch {epoch + 1}/{epochs}\n------------------------------")
    train(train_dataloader, model, loss_fn, optimizer)  # 训练一个epoch：数据加载器、模型、损失函数、优化器
    test(test_dataloader, model, loss_fn)                # 测试模型看看效果：数据加载器、模型、损失函数

print("训练和测试完成！")

# 保存模型
torch.save(model.state_dict(), "model.pth")     # model.state_dict() 是模型的参数字典，保存到 model.pth 文件中
print("模型已保存到 model.pth")

第七章：常见问题与避坑指南

1. CUDA out of memory：显存不足

  • 减小 batch_size，比如从 32 改成 16

  • 测试的时候用 with torch.no_grad()，关掉梯度计算

  • 用 torch.cuda.empty_cache() 清理没用的缓存

2. 训练不收敛，损失不动

  • 检查学习率是不是太大或者太小

  • 检查有没有忘记梯度清零 optimizer.zero_grad()

  • 检查数据预处理有没有问题，比如有没有归一化

3. Windows 下 DataLoader 卡死

  • 把你的主程序代码放在 if __name__ == '__main__': 里面，因为 Windows 的多进程需要这个
